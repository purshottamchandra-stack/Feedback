# Contributing Guidelines

Thank you for your interest in contributing to the ADD247 Faculty Feedback Form project!

## Development Setup

1. Clone the repository:
```bash
git clone https://github.com/YOUR_USERNAME/ADD247-Faculty-Feedback-Form.git
cd ADD247-Faculty-Feedback-Form
```

2. Install dependencies:
```bash
npm install
```

3. Authenticate with Google:
```bash
npm run login
```

## Making Changes

1. Create a feature branch:
```bash
git checkout -b feature/your-feature-name
```

2. Make your changes in the `src/` directory

3. Test locally:
```bash
npm run push
```

4. Verify the web app works correctly

5. Commit your changes:
```bash
git add .
git commit -m "Add: Clear description of changes"
```

6. Push to your fork:
```bash
git push origin feature/your-feature-name
```

7. Open a Pull Request with a detailed description

## Code Style

- Use clear variable names
- Add comments for complex logic
- Follow existing code formatting
- Test all changes before submitting

## Reporting Issues

- Check existing issues first
- Provide clear steps to reproduce
- Include browser/environment info
- Attach screenshots if applicable

## Pull Request Process

1. Update documentation if needed
2. Add comments for new functions
3. Test on multiple devices/browsers
4. Request review from maintainers
5. Address feedback and iterate

## Future Enhancements

Thank you for contributing!
